# 00 - Create Setup Scripts

```text
ROLE:
You are a senior QA automation engineer. You create setup scripts for an external Playwright + TypeScript E2E harness.

GOAL:
Create or refresh the setup scripts that let a beginner create an external AI test harness workspace from only two URLs.
The scripts may provide course example defaults, but the generated files must stay generic and must not assume any application feature names.

CONTEXT:
Run this prompt from the course repo, inside modulo-07-ia-mcp.
The generated harness must live outside the course repo so it stays separated from the SUT and from course material.

FILES TO CREATE OR UPDATE:
1. scripts/setup-ai-harness.ps1
   - Windows PowerShell version.
2. scripts/setup-ai-harness.sh
   - macOS/Linux bash version.

SCRIPT REQUIREMENTS:
- Accept a required target directory:
  - PowerShell: -TargetDir <path>
  - Bash: <target-dir>
- Accept optional target URLs with course example defaults:
  - PowerShell: -UiUrl <url> -ApiUrl <url>
  - Bash: --ui-url <url> --api-url <url>
- Accept a force option for a non-empty target:
  - PowerShell: -Force
  - Bash: --force
- Accept an option to avoid opening VS Code:
  - PowerShell: -NoOpenCode
  - Bash: --no-open-code
- Validate these tools:
  - node
  - pnpm
  - claude
  - git
  - gh as optional
  - code as optional
- If pnpm is missing and corepack exists, enable pnpm through corepack.
- Create the target directory if it does not exist.
- Stop if the target directory is not empty unless force is enabled.
- Create:
  - .mcp.json for Claude Code
  - .vscode/mcp.json for VS Code
  - prompts/ copied from modulo-07-ia-mcp/prompts
  - PROJECT_BRIEF.md
  - CLAUDE.md
  - .gitignore
- Initialize git in the target if needed.
- Open VS Code only when code exists and the no-open option was not used.
- Never create src/, package.json, playwright.config.ts, or feature files. Those are created later by the architect prompt.

PROJECT_BRIEF.md CONTENT:
Write a short, stable brief with:
- mission
- target UI and API URLs
- toolchain
- architecture goals
- quality rules
- token efficiency rules
Include that the AI must create TEST_PLAN.md after the foundation and before feature implementation, with UI/API cases and confirmed endpoints.

CLAUDE.md CONTENT:
Write short project instructions for Claude Code:
- read PROJECT_BRIEF.md first
- read AGENTS.md when it exists
- read TEST_PLAN.md before creating slices
- do not create app-specific code before TEST_PLAN.md
- make small verified changes
- do not commit secrets or generated reports

MCP CONFIG:
Use Playwright MCP through npx:
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    }
  }
}

QUALITY RULES:
- Keep scripts readable for beginners.
- Prefer clear errors over silent behavior.
- Do not install global tools silently.
- Keep generated files generic enough to adapt to any SUT by changing UI_URL and API_URL from script parameters.
- Use ASCII only.

VERIFY:
- For PowerShell, parse the script if PowerShell is available.
- For Bash, run: bash -n scripts/setup-ai-harness.sh if bash is available.
- Report the exact commands to run on Windows and on macOS.

DELIVERY:
Return:
- files changed
- verification results
- Windows command
- macOS command
- one sentence explaining that PROJECT_BRIEF.md and CLAUDE.md are generated by the setup script in the external harness folder
```
